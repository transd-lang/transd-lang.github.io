About Transd

What is Transd?

Transd is a programming language implemented with the meta-compilation technology which allows the programs written on Transd to run faster than on other non-compiled languages and in the same time not to employ the real compilation to machine code.

What was the motivation for its creation?

We needed an extension language which was 1.fast; 2.compactly implemented; 3.not compiled to machine code.

How fast is Transd?

Depending on the computational patterns of the code, the programs written on it run on the average from 50% to 80% of native speed.

What are the costs of this efficiency?

Increased usage of memory. Transd programs need considerably more memory for running than their native analogs.

How compact is Transd?

Currently it can be added as a library to a C++ program in the form of two files: "transd.hpp" and "transd.cpp". At the time of this writing (May, 2021) these two files have in sum about 12 000 lines of code. It is a design goal that this number remained well under 30 000.

Is it ready for using?

Not for production. Presently (May, 2021), it's in alpha stage and being readied for the beta version. Early adopters are welcome to try it.

What is the Transd's license?

Its source code C++ library (files 'transd.hpp' and 'transd.cpp') can be used freely for any purposes with the sole condition that the source code remain unchanged. On the early stage of language usage we want to prevent the proliferation of incompatible dialects of Transd.

The coding and naming styles in 'transd.cpp' and 'transd.hpp' files look unusual. Is the code obfuscated?

No, the code is not obfuscated. It's minimized. It's a design goal to keep the embeddable Transd library compact and contained just in two files. Because compilers have limitations on the maximal size of a source file, it's necessary to minimize the length of identifiers and strip out the empty lines.

Will the source code be published?

Yes, if the language gains a sufficient number of users and there will be sense to put an amount of work into reorganizing the source code into a public code base.

About author

Transd language and its meta-compiler (Tourbillon) were created by Albert Berger in 2020, after his idea that computation can be efficiently implemented with a finite-automata network has been successfully tested in the end of 2018.
