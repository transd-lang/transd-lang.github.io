Development

The leading edge of Transd development can be observed in two repositories:

TREE3 - Transd interpreter in which the latest versions of Transd embeddable library are incorporated as they are ready. Once a month, a current release is made. To get the latest features and bug fixes it's recommended to build the current version of TREE3 from the current source code in repo.

Test Suit - Transd reference test suite, which represents the current state of the language. All new language features and syntax variants appear first in the test suite.

Contribution

Transd's development is currently driven by an established set of goals and design decisions of its authors. Source code of the language implementation is currently available only in the form of minimized embeddable library. Contribution is welcomed in the form of bug reporting, spreading the word, and ideas about additional features of the language.

Current tasks:

Arbitrary precision numeric types

In addition to the existing fixed precision numeric types - Double, Int, Long, ULong - largest of which is 64-bit wide, a new integer type BigLong is currently programmed, which will have arbitrary precision. As a more distant goal - the arbitrary precisioned rational number type is planned.

Release of version 1.0

After testing and debugging the beta-version, the release of production-ready version 1.0 of Transd is planned.
Estimated completion time: First quarter of 2023.
