Lexical conventions




Transd source files are encoded in UTF-8. Upper and lower character cases are distinguished.

Source files can contain comments, which can be marked by one of two ways:

1. Character sequence "//” (slash slash) anywhere on a line marks the start of a comment which extends to the end of the line.
2. Character sequence “/*” (slash asterisk) marks the start of a comment which extends to the nearest character sequence “*/” (asterisk slash).

Identifiers in Transd can be any string of letters, digits, and underscores, not beginning with a digit. Letter is any symbol considered alphabetic under the current locale.

The following keywords are reserved for internal use in Transd and cannot be used as identifiers by the user:

and or for while break
throw continue where do not
if else elsif true false

There are several predefined built-in variables in Transd. The names of built-in variables begin with ‘@’. Since the names of user defined variables cannot contain this character, there cannot be name clashes with built-in variables.
