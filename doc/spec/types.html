Types



Transd is a strongly/statically typed language. All data variables and function parameter must have an associated type. In assignment operations types of both operands must be compatible. Types of arguments passed to a function must be compatible with the formal parameters in the function declaration.

Transd has two kinds of types: built-in types and user types. All the type discipline rules apply uniformly to both kinds.

User types are created in Transd programs by defining new classes.

Built-in types can be grouped into two categories: fundamental (or basic) types and auxilliary types, which also could be called built-in classes.

Transd has the following fundamental types:

Null, Bool, Int, Double, String, Vector, Index.

Below is the summary of statements with which objects of basic types can be declared in a Transd program:

a: Null(),       // null

a: true,         // bool

a: Bool(true),   // bool

a: 5,            // integer

a: Int(5),       // integer

a: 5.0,          // double

a: Double( 5.0 ),  // double

a: “Hello, world!”,  // string

a: String( “Hello, world!” ),  // string

a: [1, 2, 3],    // vector of integers

a: Vector( Int() ), //empty vector of integers

a: Vector( Int() 3), // vector of integers [0, 0, 0]

Index a: { a: 25, b: 36, c: 47 }, // index

a: Index( String() Int() ), // index with strings as keys and integers as values




Built-in classes include:

Range: a class for defining iteration intervals and increments for using in iteration constructs (such as ‘for’ statement).

Directory: a class for enumerating files in a file directory according to a file name pattern.
