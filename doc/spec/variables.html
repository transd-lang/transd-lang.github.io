Variables



Variable here means a name associated with a data object. Variables in the program appear as declarations of either class/module data fields, or local variables in statements:

module1: {
var1: 15, // var1 - data field

}


All variable declarations in Transd are simultaneously definitions, possibly with default values. So there cannot be untyped variables as well as uninitialized variables. The required storage for a variable is allocated at the place of its declaration. The deallocation of the storage is done by garbage collector as and when necessary.
