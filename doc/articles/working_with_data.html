Using Transd for working with data

Transd's data processing capabilities include a built-in data query language (TQL) for working with datasets in a database way, and data deserializing functionality which makes it possible to create program objects and initialize them with data directly from text data files.

Transd Query Language

Transd Query Language (TQL) is a built-in tool in Transd for working with datasets of various types by making database-like queries for selecting and updating data. TQL can be used for working with the following dataset types:

  1. Tables - tabular data with values organized into rows and named columns, such as data in CSV format, etc.: 
      ID        Name       Salary 
 234   Edward Smith     60000
 235   Harriet Willers  70000
  2. Collections of records - data with values organized into records: groups of named fields: 
    Device: Tablet
Screen: 10.1
Weight: 350

Device:   Notebook
RAM (Gb): 16
HD (Tb):  3
  3. Weakly structured data - a collection of text entries (documents) that have some header or tags attached.

Example of a TQL 'select' query:


(tql table1 
    select: ["Book_title", "Price"]
    as: [[String(), Double()]]
    where: "Price < 50"
    satisfying: (lambda Author String() (match Author ".*Lovecraft.*"))
    sortby: "Price" :ascending
    limit: 5 
)

Data deserializing

Data deserializing in Transd is a mechanism of initializing the program state using data and configuration parameters provided in a text string (e.g. from a file). This mechanism is similar to the deserializing of JSON data into program objects in such languages as Java, JavaScript, C#, etc. In Transd data deserizalizing offers additional essential features such as the ability to use variables as field values, that refer to objects in the same data file, the ability to have objects of other classes as field values, and so on. In Transd for data deserializing the Transd Structured Data (TSD) format is used, which resembles JSON, but has a number of differences.
