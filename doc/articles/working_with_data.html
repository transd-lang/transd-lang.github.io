Using Transd for Working with Data

The data related functionality in Transd subdivides into two areas:

  1. Data processing
  2. Data importing

Data processing means the tasks where Transd is used as a tool for querying and modifying an external dataset.

Data importing means the cases where data is imported into a Transd program and becomes an active part of the program.

This page after a short introduction describes three practical examples of using Transd for working with data: two examples for data processing and one for data importing.

Data processing

Transd can be used for working with various types of datasets. These types can be classified into three categories:

  1. Tables
  2. Collections of records
  3. Loosely organized data aggregates

Tables

Tables is the most structured dataset type. All values in table datasets are logically grouped into rows and columns. Transd defines the Table type, which represents the table data structure. Rows in a Table object can have differently typed fields. Fields of different records are combined into named columns. Each column can contain values of one of the following types: String, Int, Double, DateTime.

Transd can read and load table datasets stored in one of the several variations of CSV format. Other formats, with some conversions, can be used too.

Collections of records

A record is a structured description of some object. A record, can have differently typed named fields, describing individual properies of the object. A collection of records is represented in Transd by the TSDBase data type. The difference of the TSDBase class from Table class is that records in a TSDBase object, unlike rows in a table, can have different sets of fields.

A dataset with a collection of records can be stored as a set of TSD objects, that is chunks of data in TSD format (TSD is short for "Transd Structured Data"). TSD format resembles JSON with some additions, such as distinguishing between floating and integer numbers, including arrays as object members, etc. (The source code of any Transd program is precisely a collection of records, with program modules being records in TSD format.)

Data aggregates

A data aggregate is an assembly of loosely structured data, whose individual data chunks (entries) have some classifiers, headers or tags (meta data), allowing to group these chunks or connect them with each other into a hierarchical structure.

If the meta data in individual entries allows to connect them into a single logical structure, then big collections of seemingly assorted data entries can organized into a TSDBase database and queried in a regular way.

This way of handling data is suited for working with knowledge databases, personal notes, assorted ideas, etc.

TSD Query Language

The central part of data processing in Transd is TSD query language - TQL (pronounced 'tee-quel'). All data processing tasks can be grouped into three main types:

  1. Selecting data
  2. Modifying data
  3. Doing calculations on data

TQL defines for these types of tasks three types of queries:

  1. SELECT
  2. UPDATE
  3. REDUCE

TQL queries are performed uniformly on any kind of supported data structure. That is, a table, a TSD database, or even a vector of values, can be queried using the same syntax of queries.

Example of a TQL 'select' query:


(tql table1 
    select: ["Book_title", "Price"]
    as: [[String(), Double()]]
    where: "Price < 50"
    satisfying: (lambda Author String() (match Author ".*Lovecraft.*"))
    sortby: "Price" :ascending
    limit: 5 
)

Data importing

Data importing means here converting external data into program objects. Data importing is a powerful mechanism making it possible to outsource data related functionality from the program to external means, and, using some universal data exchange format, to transfer and incorporate data objects, defined externally, directly into the program state at runtime.

Data importing can help both to reduce considerably the program size, as well as to use for handling data specialized tools most appropriate for the task.

In Transd data importing is implemented on the basis of TSD data format, which is supported at deep levels of the language.
